The Shell

The shell is a program that takes COMMANDS form the keyboard and gives them to the Operating System to perform. Usually the shell that comes installed on most linux systems is called BASH, "The Bourne Again Shell". BASH is based on an older shell called the Bourne shell. There are several other shell that are fairly similiar to bash but with some added functionality.

  • korn Shell
  • C Shell
  • Tcsh
  • Zsh
  • c shell

Users are free to change their shell using a program called Usermod.

The Terminal

What we think of as a terminal, is actually usually a terminal emulator. A terminal emulator is a program that lets you interact with a shell. In the early days of computing the only way to interact with a computer was through a terminal. A terminal wasn't a computer. It was just a keyboard and a monitor(or a printer) that could communicate with a computer, usually a main frame. The programs we call terminals that we run within our Gui's are called terminal emulators. If we set up our linux systems without GUI's then we can interact with the computer using a text based console using a text mode interface. There are many terminal programs and the user can load and install any terminal emulator they would like. Please learn the keyboard short cut to open your terminal. Many terminal programs allow you a tab systems so you can open many terminals at once and move between each easily and freely.

Root User

The root is the administrator acount in Linux, the root user has elevated privelges not usually given to regular users.

Command Line Syntax

A computers operation can generally be described in 3 steps

  1. Computer waits for user input.
  2. User selects a command and enters it with keyboard or mouse
  3. Computer executes the command.

When you use a terminal, the shell program displays a prompt where users can enter commmands using the keyboard. That prompt usually has

  • user name
  • followed by the @ symbol
  • followed by the name of the computer
  • followed by a colon and the present working directory
  • in Bash followed by a $

as seen below 

user@host:~$

Commands

Are a sequence of characters in a line which ends by pressing the enter key. When you press enter the command is evaluated by the shell.

command syntax

To interpert a command the shell breaks the line you typed in down to indiviual tokens separated by spaces. There are 3 main categories of what the shell looks for in a line passed as a command.

  1. Command Name (Command Name)
  2. Command Switches, these start with a dash -
    • Command switches turn on and off different options for the program your calling These can be written as -l -a -s or -las, somtimes long options are allowed as well for readability like --switch-on. (OPTIONS)
  3. Command Arguments -> usually file names (ARGUMENTS)

Shell Scripts

The Shell lets us simplify repetitive and Huge tasks by writing shell scripts. A shell script is simply a file containing a sequence of commands. The shell is going to open the file and execute the command as if you were entering them one at a time by yourself.

Variables

An essential feature of a Shell is the ability to use variables. A variable can hold a value or a command. Variables are areas of memory that can be used to store information and are referred to by a name. When ever the shell see's the dollar sign and a NAME it considers it a variable and tries to find out the value of that variable and use that value in place of the name. 

$NAME

To create a variable a line is place in the script,or at the command prompt that has a name followed by and equal sign followed by a value with NO SPACES. 

age=40
name=ProfessorZ

echo $name //will output ProfessorZ to std out
Rules for naming variables
  • Var names must start with a letter
  • a name must not contain embedded spaces(you can use underscores)
  • punctuation marks are not allowed

Environment Variables

When your shell session starts many variables are already set by a startup file. to see which variables are set for the user environment use the printenv command for print environment. This will print all the environment variables to the screen. By convention environment variable are UPPERCASE.

Meta Characters

When the shell reads the input, it reads the command name, the option switches, and the arguments then runs the command. There are certain characters that the shell treats as special. If we want to stop the shell from acting on and expanding meta characters we have to quote them by using either \ or ' ' or " ".

  • " " Separates arguments
  • *
  • ; separates commands allows you to do multiple commands on one line.
  • >
  • ?
  • &
  • | pipe

Examples

if you want to use any of the above characters as regular characters instead of as Meta characters you would put a \ in front of it. 

echo Hello \*World\*

// will print  Hello *World*

ls -la /home/me
// command is ls, options or -la , argument is /home/me

//ls -la prints out
//permissions owner group file_size and other information

if you don't us the \ the * will act as a meta character and you will get different results.
